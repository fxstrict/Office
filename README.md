## 🌟 نظرة عامة

نظام **إدارة مكاتب المحاماة** هو تطبيق ويب احترافي متكامل مصمم خصيصاً لمكاتب المحاماة العربية. يوفر النظام إدارة شاملة للقضايا والموكلين والجلسات والمستندات مع تكامل كامل مع خدمات Google Workspace.

### 🏗️ البنية التقنية

```
Frontend          →  Next.js 15 + React 19 + TypeScript + Tailwind CSS
Backend / قاعدة البيانات  →  Google Apps Script + Google Sheets
التخزين          →  Google Drive
التقويم          →  Google Calendar
البريد           →  Gmail API
```

## 🚀 دليل التركيب والتشغيل

### الخطوة 1: إعداد Google Apps Script (الباك إند)

#### 1.1 إنشاء مشروع جديد
1. اذهب إلى [script.google.com](https://script.google.com)
2. اضغط **"مشروع جديد"**
3. أعطِ المشروع اسماً: `Law Office Management`

#### 1.2 رفع ملفات الباك إند
انسخ محتوى هذه الملفات إلى المشروع:

**الملف الأول** - في Apps Script، اضغط `+` بجانب الملفات واختر "Script":
- اسمه: `Initialize`
- المحتوى: من `backend/gas/01_Initialize.gs`

**الملف الثاني** - أضف Script جديد:
- اسمه: `API`
- المحتوى: من `backend/gas/02_API.gs`

**ملف الإعداد** - عدّل `appsscript.json`:
- في Apps Script اضغط على ⚙️ ثم "إعدادات المشروع"
- فعّل "إظهار ملف appsscript.json"
- انسخ محتوى `backend/gas/appsscript.json`

#### 1.3 تهيئة النظام لأول مرة
1. في Apps Script اختر دالة `initializeSystem`
2. اضغط **تشغيل** ▶️
3. وافق على الصلاحيات المطلوبة
4. انتظر رسالة النجاح في السجل

#### 1.4 نشر السكريبت كـ Web App
1. اضغط **"نشر"** ← **"نشر جديد"**
2. اختر النوع: **"تطبيق ويب"**
3. الإعدادات:
   ```
   تنفيذ بوصفك: أنت (Me)
   من يمكنه الوصول: أي شخص (Anyone)
   ```
4. اضغط **"نشر"**
5. **انسخ الرابط الظاهر** - ستحتاجه في الخطوة التالية

> ⚠️ **مهم**: احفظ رابط الـ Web App - يبدأ بـ `https://script.google.com/macros/s/...`

#### 1.5 إعداد المشغّلات التلقائية
1. في Apps Script اختر دالة `setupTriggers`
2. اضغط **تشغيل** ▶️
3. هذا سيُنشئ:
   - مشغّل كل ساعة لإرسال التذكيرات
   - مشغّل منتصف الليل للنسخ الاحتياطي

### الخطوة 2: إعداد الفرونت إند (Next.js)

#### 2.1 تنزيل وتثبيت المشروع

```bash
# انتقل إلى مجلد الفرونت إند
cd law-office-system/frontend

# ثبّت الحزم
npm install

# أو باستخدام yarn
yarn install
```

#### 2.2 إعداد متغيرات البيئة

```bash
# انسخ ملف المثال
cp .env.example .env.local
```

افتح `.env.local` وعدّل:
```env
NEXT_PUBLIC_GAS_URL=https://script.google.com/macros/s/YOUR_SCRIPT_ID_HERE/exec
```

> استبدل `YOUR_SCRIPT_ID_HERE` برابط الـ Web App الذي نسخته في الخطوة 1.4

#### 2.3 تشغيل بيئة التطوير

```bash
npm run dev
```

افتح المتصفح على: **http://localhost:3000**

#### 2.4 بناء النسخة الإنتاجية

```bash
# بناء المشروع
npm run build

# تشغيل الإنتاج محلياً
npm start
```

### الخطوة 3: النشر على الإنترنت

#### خيار 1: Vercel (الأسهل والأسرع - مجاني)

```bash
# ثبّت Vercel CLI
npm install -g vercel

# سجّل الدخول
vercel login

# انشر المشروع
vercel --prod
```

أو عبر واجهة Vercel:
1. اذهب إلى [vercel.com](https://vercel.com)
2. اضغط **"New Project"**
3. ارفع مجلد `frontend` أو اربطه بـ GitHub
4. أضف متغير البيئة: `NEXT_PUBLIC_GAS_URL`
5. اضغط **"Deploy"**

#### خيار 2: Netlify

```bash
npm install -g netlify-cli
netlify deploy --prod --dir=.next
```

#### خيار 3: GitHub Pages + GitHub Actions

أضف هذا الملف: `.github/workflows/deploy.yml`
```yaml
name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: 18
      - run: npm ci
      - run: npm run build
        env:
          NEXT_PUBLIC_GAS_URL: ${{ secrets.GAS_URL }}
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./out
```

### إنشاء قضية جديدة

1. من الشريط الجانبي اضغط **"القضايا"**
2. اضغط **"قضية جديدة"** (الزر الأخضر)
3. أدخل البيانات:
   - رقم القضية (مطلوب)
   - السنة (مطلوب)
   - المحكمة (مطلوب)
   - نوع القضية (مطلوب)
   - موضوع القضية (مطلوب)
4. اختر الموكلين المرتبطين (Ctrl + Click للاختيار المتعدد)
5. اضغط **"إنشاء القضية"**

---

### إضافة موكل ورابط البوابة

1. اضغط **"الموكلون"** من القائمة الجانبية
2. اضغط **"موكل جديد"**
3. اختر نوع الموكل (فرد / شركة)
4. أدخل البيانات الكاملة وأضف بريده الإلكتروني
5. اضغط **"إضافة الموكل"**
6. في صفحة الموكل، اضغط **"إنشاء بوابة وQR Code"**
7. سيُرسل QR Code تلقائياً لبريد الموكل

---

### إضافة جلسة مع تذكيرات تلقائية

1. افتح صفحة القضية المطلوبة
2. اضغط تبويب **"الجلسات"**
3. اضغط **"إضافة جلسة"**
4. أدخل التاريخ والوقت والقرار
5. اضغط **"إضافة الجلسة"**

**سيتم تلقائياً:**
- ✅ إنشاء حدث في Google Calendar
- ✅ إضافة تذكيرات: 7 أيام، 3 أيام، 24 ساعة، ساعة

---

### إدارة المكتبة القانونية

1. اضغط **"المكتبة القانونية"** من القائمة
2. لإضافة رابط: اضغط **"إضافة"**
3. أدخل: الأيقونة، العنوان، الرابط، الوصف، التصنيف
4. اضغط **"إضافة"**

للنماذج الديناميكية: اضغط تبويب **"نماذج الدعاوى"** وأضف الأزرار التي تريد.

---

### إنشاء نسخة احتياطية

**تلقائياً:** يومياً / أسبوعياً / شهرياً (حسب الإعدادات)

**يدوياً:**
1. اذهب إلى **"الإعدادات"**
2. اضغط قسم **"النسخ الاحتياطي"**
3. اضغط **"إنشاء نسخة احتياطية الآن"**

## 🔒 الأمان والحماية

### تشفير كلمات المرور
```
SHA-256(password + "LawOfficeSalt2024") → passwordHash
```

### إدارة الجلسات
- Token عشوائي 64 حرف
- مخزن في Google Apps Script Cache
- ينتهي بعد 8 ساعات من عدم النشاط

### بوابة الموكل
- UUID عشوائي 64 حرف
- لا يحتوي على معرّف الموكل
- قابل للإلغاء فوراً
- له تاريخ انتهاء قابل للتعديل

### أمان الـ API
- كل طلب (إلا العامة) يحتاج Token صالح
- التحقق من الدور لكل endpoint
- تسجيل كل العمليات في سجل النشاط

## 🐛 حل المشكلات الشائعة

### المشكلة: "خطأ في الاتصال بالخادم"
**الحل:**
1. تأكد من صحة `NEXT_PUBLIC_GAS_URL` في `.env.local`
2. تأكد أن الـ Web App منشور بصلاحية **"أي شخص"**
3. أعِد نشر الـ Web App وانسخ الرابط الجديد

### المشكلة: "غير مصرح - يرجى تسجيل الدخول"
**الحل:**
1. امسح بيانات المتصفح (Local Storage)
2. سجّل الدخول من جديد

### المشكلة: "فشل تهيئة النظام"
**الحل:**
1. تأكد من قبول جميع الصلاحيات عند تشغيل `initializeSystem`
2. تأكد أن حساب Google يملك Google Drive

### المشكلة: "جلسات Google Calendar لا تُضاف"
**الحل:**
1. تأكد أن صلاحية Google Calendar موجودة في `appsscript.json`
2. شغّل `initializeSystem` مجدداً وقبل الصلاحيات
